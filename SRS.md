# เอกสารข้อกำหนดซอฟต์แวร์ (SRS): การผจญภัยของซาร่า (Sara's Adventure)

## 1. บทนำ
"การผจญภัยของซาร่า" (Sara's Adventure) เป็นเกมแนวผจญภัยแบบ 2D Top-down ที่พัฒนาด้วยภาษา Python และ Pygame โดยเน้นไปที่การสำรวจ การเคลื่อนที่ของตัวละคร และการแสดงผลแอนิเมชันผ่านแผนที่ที่มีความหลากหลาย

## 2. ภาพรวมของโครงการ
เป้าหมายของเกมคือการควบคุมตัวละครชื่อ "ซาร่า" (Sara) เดินทางผ่านแผนที่ 2 รูปแบบเพื่อค้นหาถ้วยรางวัลแห่งชัยชนะ เกมเน้นความลื่นไหลของการควบคุมและการตอบสนองต่อเงื่อนไขในเกม

## 3. ข้อกำหนดด้านฟังก์ชัน (Functional Requirements)

### 3.1 การเคลื่อนที่ของตัวละคร
- **การควบคุม**: ผู้เล่นสามารถบังคับซาร่าไปใน 4 ทิศทาง (บน, ล่าง, ซ้าย, ขวา) โดยใช้ปุ่มลูกศรบนคีย์บอร์ด
- **ความเร็ว**: การเคลื่อนที่ต้องมีความแม่นยำและลื่นไหล (กำหนดไว้ที่ 5 พิกเซลต่อก้าว)
- **ขอบเขต**: ตัวละครจะไม่สามารถเดินออกนอกขอบเขตของหน้าจอเกมได้ (400x300 พิกเซล)

### 3.2 แอนิเมชันของตัวละคร
- **การแสดงผล**: ซาร่าจะมีการเปลี่ยนท่าทาง (Animation) ตามทิศทางการเคลื่อนที่
- **ระบบ Spritesheet**: ใช้ไฟล์ภาพ Spritesheet ในการจัดการเฟรมต่างๆ ของตัวละคร

### 3.3 ระบบแผนที่และระดับ
- **ขนาดแผนที่**: แผนที่แต่ละฉากจะมีพื้นที่ขนาด 20x20 ไทล์ (Tiles)
- **สถาปัตยกรรมแบบ Layer**: รองรับการแสดงผลแยกจากกัน 3 ชั้น:
    - **ชั้นพื้นหลัง (Ground Layer)**: สำหรับพื้นผิวพื้นฐาน (หญ้า, ดิน)
    - **ชั้นเส้นทาง (Path Layer)**: สำหรับถนน หรือสิ่งตกแต่งพื้นผิว
    - **ชั้นวัตถุ (Item Layer)**: สำหรับสิ่งของที่โต้ตอบได้ (ประตูมิติ, ถ้วยรางวัล)
- **แผนที่ที่ 1 (Forest)**:
    - ใช้ชุดภาพประกอบฉากแบบป่า (`forest_tileset.png`)
    - มี **ประตูมิติ (Portal)**: เมื่อซาร่าเดินไปสัมผัส จะถูกส่งไปยังแผนที่ที่ 2
- **แผนที่ที่ 2 (Space)**:
    - ใช้ชุดภาพประกอบฉากแบบอวกาศ (`space_tileset.png`)
    - มี **ถ้วยรางวัล (Gold Trophy)**: วางอยู่ในตำแหน่งที่กำหนดเพื่อให้ซาร่าไปเก็บ

### 3.4 เงื่อนไขการชนะ
- **ชัยชนะ**: เมื่อซาร่าเก็บถ้วยรางวัลในแผนที่อวกาศได้ เกมจะแสดงข้อความแสดงความยินดี ("Congratulations!") และจบเกม

## 4. ข้อกำหนดด้านส่วนต่อประสานกับผู้ใช้ (UI Requirements)
- **หน้าต่างเกม**: ขนาด 400x300 พิกเซล
- **หัวข้อหน้าต่าง**: "Sara's Adventure"
- **ไอคอนหน้าต่าง**: ใช้ไอคอนที่กำหนดเอง (`my_icon.png`)

## 5. ข้อกำหนดที่ไม่ใช่ฟังก์ชัน (Non-Functional Requirements)
- **ประสิทธิภาพ**: เกมต้องทำงานที่ความเร็ว 144 FPS อย่างเสถียร
- **ระบบเสียง**: ไม่มีการใช้งานระบบเสียงในเวอร์ชันนี้

## 6. เทคโนโลยีที่ใช้
- **ภาษา**: Python
- **ห้องสมุด**: Pygame
- **ทรัพยากร**: ไฟล์ภาพ PNG สำหรับตัวละคร, ฉาก, ไอเทม และไอคอน
